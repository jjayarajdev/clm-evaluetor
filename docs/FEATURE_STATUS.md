## 1. AI Agents (10 agents)

The platform has 10 AI agent files in `backend/app/agents/`. CLAUDE.md says 9 — the discrepancy is that `intent_router.py` is a routing helper, not a standalone agent.

All agents share `base.py` which provides the `AsyncOpenAI` client + RAG tooling, and route through `services/orchestrator.py:route_request()` for tracing/Langfuse. DSPy-compiled programs in `services/dspy_extractor.py` are an orthogonal optimization path — used when a tenant has a compiled program, otherwise falls back to the orchestrator.

**Reality of the upload pipeline (end-to-end):**

When a contract uploads via `POST /api/contracts/upload`, this is what actually runs:

```
Indexing phase (indexer.py):
  → parse_file (PDF/DOCX text extraction)
  → chunk + embed in ChromaDB (vector store)
  → metadata extraction (real LLM call, persisted)
  → optional: custom field extraction (if tenant has definitions)
  → risk assessment (real LLM, chunked, persisted)
  → contract reference extraction (real LLM, persisted)
  → mark contract COMPLETED
  → invalidate dashboard cache (non-fatal)
  → hierarchy detection (limited to 50 most recent contracts)
  → auto-link detection (suggested_links — non-fatal)

Deep analysis phase (contracts.py:_run_deep_analysis, async):
  → re-parse document
  → load few-shot examples per tenant
  → load industry-specific hints
  → clause extraction (real LLM, persisted)
  → reclassify SLA chunks (semantic re-tag)
  → obligation extraction (real LLM, persisted)
  → SLA extraction (Excel first, then LLM, persisted)
  → highlight extraction (PDF rects for UI — non-fatal)
  → taxonomy discovery (suggests custom taxonomy — non-fatal)
  → renewal analysis (real LLM, persisted)
  → schema extraction (per-contract-type structured fields — non-fatal)
  → compliance gap detection
  → regulatory obligation extraction (regulated industries only)
  → knowledge graph extraction (deferred from indexer to avoid FK violations)
  → governance bridge: auto-create Org + Relationship + KPIs + improvement points
```

**Silently optional / can fail without breaking the contract:**
- Custom field extraction (skipped if tenant has no definitions)
- Highlight extraction
- Taxonomy discovery
- Schema extraction (skipped if contract type doesn't match a known schema)
- Knowledge graph extraction
- Governance bridge automations

## 12. What to Demo Confidently

These flows are end-to-end real:

1. **Upload → AI extraction → contract page populated**
   Metadata, clauses, risks, obligations, SLAs, renewal terms, references, regulatory obligations (regulated industries), schema fields all extracted by real LLM calls and persisted.

2. **Compliance & Performance dashboard**
   All 6 widgets are dynamic. Click-through on any item navigates to the actual record. SLA breaches show real deviation/penalty calculations.

3. **Obligation lifecycle**
   Status updates, RAG transitions, owner assignment, evidence file upload, compliance rate aggregation.

4. **SLA performance**
   Log a measurement → automatic breach detection → penalty calculation → alert created → notification dispatched (via email).

5. **Renewals**
   Calendar of expiring contracts, notice deadline tracking, status updates (pending/approved/declined/renegotiating), ICS export.

6. **Vendor scorecards**
   Performance scoring combining obligation + SLA compliance, risk tiers, comparison.

7. **Q&A on contracts**
   "Which contracts expire in 30 days?" → structured query path. "What does this clause mean?" → RAG path with citations.

8. **External vendor portal**
   Share a contract via token, vendor logs in (no auth needed), views shared content, submits perception scores feeding KPI gaps.

9. **Governance bridge**
   Upload a vendor MSA → automatically creates Organization, BusinessRelationship, KPIs from SLAs, ImprovementPoints from high-risk clauses, HealthScore.

10. **Multi-tenant admin**
    Super-admin creates tenant, provisions users, configures industry profile, business units, SSO. Real isolation.

11. **Scheduler + alerts**
    Background job runs every 15 minutes, generates alerts, dispatches via email. (See gap: it pulls actuals from a stub.)

12. **Compliance reports + CSV export**
    Real date-range aggregation, real CSV blob download.

## 13. Known Gaps — What NOT to Demo Without Setup

Be aware of these before a live demo:

| Gap | Impact | Mitigation |
|-----|--------|------------|
| **ServiceNow / Milestone / FX connectors are stubs** | Scheduled SLA comparison runs against fake actuals; FX threshold alerts use fake rates | Either mark as "demo data" in narration, or don't drill into the source of those values |
| **SharePoint import is skeletal** | Can configure but not actually import a folder | Show config UI only; don't click Import |
| **Slack / Webhook delivery channels** | Configurable, but only email actually delivers | Use email-only notification rules in demo |
| **DSPy compilation has no UI** | Tenants benefit from compiled programs but compilation is script-driven | Pre-compile for the demo tenant; don't surface this |
| **Two dashboards exist** | `DashboardPage` is legacy; `ModernDashboardPage` is canonical | Remove legacy or hide route before demos |
| **SettingsPage tab depth uneven** | Some tabs persist, others may not | Test each tab on the demo tenant first |
| **Suggested links UI has thin spots** | Endpoint review-and-approve flow is partial | Show auto-link results on the contract page instead |
| **Extraction Quality verification UI** | Framework + bulk-approve real; per-record review UX is basic | Demo bulk-approve; avoid record-by-record review |
